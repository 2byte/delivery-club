# Варианты Развертывания Клиента

## Сравнение Вариантов

| Вариант | Размер | Зависимости | Установка на целевой машине |
|---------|--------|-------------|----------------------------|
| **1. EXE + node_modules** | ~116 МБ | node_modules (~2 МБ) | Просто скопировать папку |
| **2. BAT + Bun** | ~50 КБ | Bun runtime (~90 МБ) | Установить Bun один раз |
| **3. Портативный Bun** | ~92 МБ | Встроен | Скопировать папку |

## Вариант 1: EXE + node_modules (Рекомендуется для простоты)

### ✅ Плюсы:
- Один раз скопировал - работает
- Не нужно ничего устанавливать
- Работает как обычная программа

### ❌ Минусы:
- Нужна папка node_modules (~2 МБ)
- Больший размер пакета

### Размер:
```
Всего: ~116 МБ
├── client.exe: ~114 МБ
└── node_modules: ~2 МБ
```

### Развертывание:
```bash
# Создать пакет
create-deploy-package.bat

# Скопировать папку deploy\ на целевую машину
# Готово!
```

### Использование:
```bash
client.exe pull
client.exe push file.txt --share
```

## Вариант 2: BAT + Bun (Минимальный размер)

### ✅ Плюсы:
- Минимальный размер (~50 КБ код + node_modules)
- Легко обновлять (просто заменить .ts файл)
- Bun устанавливается один раз на машину

### ❌ Минусы:
- Требуется установить Bun
- Нужно запускать через .bat обёртку

### Размер:
```
Код проекта: ~52 КБ
├── client.ts: ~18 КБ
├── client.bat: ~500 байт
└── node_modules: ~2 МБ (jszip)

Bun (устанавливается один раз): ~90 МБ
```

### Развертывание:
```bash
# На целевой машине установить Bun (один раз)
powershell -c "irm bun.sh/install.ps1 | iex"

# Скопировать файлы
copy client.ts
copy client.bat
copy client.config.json
copy package.json

# Установить зависимости
bun install

# Использование
client.bat pull
```

## Вариант 3: Портативный Bun (Лучшее из обоих)

### ✅ Плюсы:
- Не требует установки на целевой машине
- Всё в одной папке
- Легко обновлять код

### ❌ Минусы:
- Нужно упаковать Bun runtime в папку

### Размер:
```
Всего: ~92 МБ
├── bun/: ~90 МБ (portable runtime)
├── client.ts: ~18 КБ
├── node_modules: ~2 МБ
└── run.bat: скрипт запуска
```

### Создание:
```bash
# Скачать portable Bun
# Упаковать всё в одну папку
portable-package/
  ├── bun.exe
  ├── client.ts
  ├── client.config.json
  ├── node_modules/
  ├── package.json
  └── run.bat: .\bun.exe run client.ts %*
```

## Рекомендации

### Для разовых установок: Вариант 1 (EXE + node_modules)
Просто скопировал папку - работает. Идеально для:
- Несколько компьютеров
- Простая установка
- Не хочешь возиться

### Для многих машин: Вариант 2 (BAT + Bun)
Один раз установил Bun на все машины, дальше только код обновлять. Идеально для:
- Много компьютеров
- Частые обновления
- Централизованное управление

### Для USB/сетевых дисков: Вариант 3 (Portable)
Всё в одной папке, можно запускать откуда угодно. Идеально для:
- Портативное использование
- Запуск с сетевого диска
- Нет прав администратора для установки

## Создание Портативного Пакета (Вариант 3)

Создам скрипт для автоматической сборки:

```bash
# create-portable-package.bat
# Загружает portable Bun и создаёт полностью автономный пакет
```

## Итог

**Если не хочешь node_modules:**
- Используй Вариант 2 (требует Bun на машине)
- Или Вариант 3 (portable Bun в пакете)

**Если хочешь максимальную простоту:**
- Используй Вариант 1 (EXE + node_modules ~116 МБ)
- Запускается командой: `create-deploy-package.bat`
- Просто копируешь папку deploy\ куда нужно

Папка node_modules всего ~2 МБ, это не так много. Но если критично - используй .bat обёртку с Bun.
